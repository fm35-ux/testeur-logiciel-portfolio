# 🎯 Portfolio de Testeuse Logiciel

Portfolio personnel développé à partir d'un site existant, optimisé puis personnalisé pour présenter mon profil et mes projets de testeuse logiciel.

## 📋 Contexte du Projet

Ce projet s'inscrit dans le cadre du parcours **Testeur Logiciel** d'OpenClassrooms. L'objectif : transformer un site portfolio existant — buggé et non optimisé — en une vitrine professionnelle déployée en ligne.

Le projet se déroule en **deux phases** :

- **Phase 1 — Optimisation et correction de bugs** : prise en main d'une base de code existante, résolution des bugs et amélioration des performances, de l'accessibilité et du SEO.
- **Phase 2 — Personnalisation et déploiement** : adaptation du site à mon profil (formation, compétences, projets) et mise en ligne pour le rendre accessible aux recruteurs.

Au-delà de présenter mon parcours, le portfolio démontre mes compétences directement à travers le code du site et son interface. 🚀

## 👩‍💻 Mon rôle et missions

J'interviens sur l'intégralité du projet, de la correction technique à la mise en ligne.

Mes missions principales :

- **Corriger 3 bugs** identifiés grâce aux Chrome DevTools (icônes Font Awesome non affichées, éléments dynamiques défaillants)
- **Mettre en place 3 optimisations** sur les images, les contrastes de couleurs et la hiérarchie des titres, dans le respect des bonnes pratiques d'accessibilité et de SEO
- **Améliorer l'accessibilité** du site en m'appuyant sur les recommandations de WAVE WebAIM
- **Évaluer et améliorer les performances** à l'aide de Lighthouse
- **Personnaliser le contenu** : présentation de mon profil, de ma formation, de mes compétences et d'au moins 2 projets (avec description et lien vers le code/site)
- **Déployer le site en ligne** via GitHub Pages

## 🛠️ Stack Technique

- **HTML5** / **CSS3** — structure et mise en forme du portfolio
- **JavaScript** — éléments dynamiques du site
- **Chrome DevTools** — débogage et inspection du code
- **Lighthouse** — audit de performance, accessibilité et SEO
- **WAVE WebAIM** — audit d'accessibilité web
- **Git** / **GitHub** — versionnement du code
- **GitHub Pages** — hébergement et déploiement du site statique

## 🔍 Impact pour mon profil de testeuse logiciel

Ce projet mobilise des compétences au cœur du métier de testeuse logiciel :

- **Débogage méthodique** : l'identification et la correction de bugs via les Chrome DevTools et la console reproduisent exactement la démarche d'investigation d'une anomalie en test.
- **Audit qualité** : l'utilisation de Lighthouse et WAVE pour mesurer performance et accessibilité s'apparente à une démarche de tests non-fonctionnels (performance, accessibilité, conformité aux normes).
- **Accessibilité web** : connaître les normes d'accessibilité (contrastes, hiérarchie des titres, attributs) est un axe de test à part entière, de plus en plus exigé dans les projets.
- **Optimisation et SEO** : comprendre ce qui dégrade la qualité d'une page permet de mieux cibler les critères de validation lors des recettes.
- **Déploiement** : maîtriser la mise en ligne d'un site donne une vision concrète du cycle de vie d'une application, utile pour tester en environnement de pré-production et de production.
